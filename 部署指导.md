# 文学博客网站部署指导

## 📦 项目概述
你的文学博客是一个使用 React + Vite + TypeScript 构建的静态网站项目。

## 🛠️ 编译步骤

### 1. 环境准备
确保你的电脑已安装：
- **Node.js** (版本 16 或以上)
- **pnpm** 包管理器

如果没有安装 pnpm，可以运行：
```bash
npm install -g pnpm
```

### 2. 编译项目
在项目根目录下运行：

```bash
# 进入项目目录
cd literature-blog

# 安装依赖（如果还未安装）
pnpm install

# 编译项目
pnpm run build
```

编译成功后，会在项目根目录生成 `dist` 文件夹，里面包含所有静态文件。

### 3. 编译产物说明
`dist` 文件夹包含：
- `index.html` - 主页面文件
- `assets/` - 编译后的 CSS 和 JS 文件
- `data/` - 所有的文章、书籍、小说等数据文件
- `imgs/` - 所有图片资源

## 🚀 部署到静态托管服务

### 推荐平台（国内可访问）

#### 1. Vercel（推荐）
- 访问：https://vercel.com
- 支持 GitHub 集成，自动部署
- 国内访问速度较好

**部署步骤：**
1. 将代码上传到 GitHub
2. 在 Vercel 中导入 GitHub 仓库
3. 设置构建命令：`pnpm run build`
4. 设置输出目录：`dist`
5. 点击部署

#### 2. Netlify
- 访问：https://netlify.com
- 支持拖拽部署
- 国内访问速度一般

**部署步骤：**
1. 直接拖拽 `dist` 文件夹到 Netlify
2. 或连接 GitHub 仓库进行自动部署

#### 3. 腾讯云静态网站托管
- 腾讯云控制台 → 云产品 → 网站服务 → 静态网站托管
- 国内访问速度快
- 需要实名认证

#### 4. 阿里云 OSS 静态网站
- 阿里云控制台 → 对象存储 OSS → 静态页面
- 国内访问速度快
- 需要阿里云账号

#### 5. GitHub Pages
- 免费，但国内访问可能需要 VPN
- 适合开发者使用

### 手动部署到服务器

如果你有自己的服务器：

1. **编译项目**
```bash
pnpm run build
```

2. **上传文件**
将 `dist` 文件夹内的所有内容上传到服务器的网站根目录（如 `/var/www/html/` 或 `/usr/share/nginx/html/`）

3. **配置 Web 服务器**
确保服务器支持单页应用路由（SPA），添加以下配置：

**Nginx 配置示例：**
```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

**Apache 配置示例：**
```apache
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteBase /
    RewriteRule ^index\.html$ - [L]
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule . /index.html [L]
</IfModule>
```

## 🆘 常见问题

**Q: 编译失败怎么办？**
A: 检查 Node.js 版本，确保是 16+；删除 node_modules 文件夹后重新运行 `pnpm install`

**Q: 部署后页面空白？**
A: 检查是否正确配置了 SPA 路由，确保所有文件都上传成功

**Q: 图片不显示？**
A: 检查 `imgs` 文件夹是否完整上传，确保路径正确

**Q: 想要修改内容怎么办？**
A: 修改 `src` 文件夹中的源码，然后重新编译部署
